## AI Web Studio

AI Web Studio Pro 2026は、AIを最大限に活用し、WebサイトやWebアプリケーションの開発プロセスを加速させる統合開発環境（IDE）です。アイデアの創出からコード生成、リアルタイムプレビュー、デバッグ、そしてGitHubへのデプロイまでを、この単一のプラットフォームでシームレスに実現します。直感的でモダンなインターフェースと強力なAIアシスタンスにより、開発者はより創造的で効率的なWeb制作を体験できます。

---

### 概要

このプロジェクトは、AI（Google Gemini API）と連携し、ユーザーの指示に基づいてHTML/CSS/JavaScriptコードを生成するWebベースのスタジオです。チャットインターフェースで要件を定義し、生成されたコードをCodeMirrorエディタで編集・プレビューしながら、GitHubへのインポート・デプロイまでを一貫して行えます。Service Workerによるバックグラウンド生成やIndexedDBによるデータ永続化にも対応し、快適な開発体験を提供します。

### 主な特徴

*   **AIチャットによる要件定義とプランニング（CHATタブ）**
    *   Google Geminiモデル（設定可能）と対話しながら、Webアプリケーションの要件を具体化します。
    *   画像ファイル（カメラ、ギャラリー、ファイル）の添付に対応し、UIイメージを共有できます。
    *   メッセージの編集・削除機能で、会話履歴を柔軟に管理。
    *   プランが固まったら、「⚡️コードを作成」ボタンでAIにコード生成を指示します。
*   **統合コードエディタ（CODEタブ）**
    *   CodeMirror 6を搭載し、HTML/CSS/JavaScriptのシンタックスハイライト、コード編集、検索機能をサポート。
    *   GitHubからのコードインポート（📥 Import）やローカルファイルのアップロード（Upload）が可能。
    *   現在のコードをダウンロード（Download）する機能。
    *   GitHubへのデプロイ（🚀 GitHub Deploy）機能を統合。
*   **リアルタイムプレビューとデバッグ（PREVIEWタブ）**
    *   編集中のコードがiframe内でリアルタイムに表示され、即座に変更を確認できます。
    *   プレビュー画面下部のデバッグパネルから、AIにコードの修正依頼を送信し、改善サイクルを高速化。
*   **柔軟なAIモデル設定（SETタブ）**
    *   チャット用（PLAN）とコード生成用（BUILD）で異なるGeminiモデル（例: Flash, Pro, Deep Researchなど）を選択可能。
    *   自身のGoogle Cloud APIキーを設定することで、より高性能なモデルや最新のモデルを利用できます。APIキー未設定時もProxy Worker経由で利用可能。
    *   開発者プロファイルを設定し、AIに自身の開発スタイル（例: Tailwind CSSの使用、日本語コメント、モダンなダークモード）を指示できます。
*   **GitHub連携機能**
    *   **Import**: GitHubリポジトリ一覧からファイルを選択し、エディタに直接インポートできます。
    *   **Deploy**: 現在のコードをGitHubリポジトリにプッシュ。新規リポジトリの作成、既存リポジトリへのコミット、README.mdの自動生成、GitHub Pagesの有効化に対応しています。
*   **堅牢なデータ永続化とバックグラウンド処理**
    *   チャット履歴、現在のコード、設定、GitHubトークンなどはIndexedDBにローカル保存されるため、ブラウザを閉じても作業内容が失われません。
    *   Service Workerを活用し、AIによる大規模なコード生成をバックグラウンドで実行。生成中に他のタブに切り替えても処理は継続され、完了時には通知されます。
*   **モダンなUI/UX**
    *   Obsidian Terminal風のダークテーマを採用し、目に優しい開発環境を提供。
    *   モバイルフレンドリーなレスポンシブデザインと、スワイプによるタブ切り替えに対応。
    *   テキストエリアの自動拡張、トースト通知など、細やかなUX改善が施されています。

### 技術スタック

*   **フロントエンド**: HTML, CSS, JavaScript (Vanilla JS)
*   **AI連携**: Google Gemini API (またはProxy Worker)
*   **コードエディタ**: CodeMirror 6
    *   `@codemirror/view`
    *   `@codemirror/state`
    *   `@codemirror/lang-html`
    *   `@codemirror/theme-one-dark`
    *   `@codemirror/search`
*   **データ永続化**: IndexedDB
*   **バックグラウンド処理**: Service Worker
*   **フォント**: Google Fonts (`Syne`, `DM Sans`, `JetBrains Mono`)
*   **UI/UX**: カスタムCSS変数によるダークテーマ、レスポンシブデザイン

### セットアップ

このプロジェクトは単一のHTMLファイルで構成されており、特別なビルドプロセスや外部依存関係（CDNを除く）はありません。

1.  `index.html` (またはこのHTMLコードを含むファイル) をWebブラウザで開きます。
2.  Service Workerが自動的に登録され、バックグラウンドでのAIコード生成がサポートされます。

### 使い方

#### 1. 初期設定（SETタブ）

1.  アプリケーション上部の**「SET」タブ**をクリックして設定画面を開きます。
2.  **API KEY**: 自身のGoogle Cloud APIキー（Gemini APIを有効にしたもの）を入力します。未設定の場合でもProxy Worker経由でAI機能を利用できますが、モデル選択肢が限定されます。
3.  **GITHUB TOKEN**: GitHub連携機能を利用する場合、Personal Access Token（`repo` スコープを付与したもの）を入力します。これにより、リポジトリのインポートやデプロイが可能になります。
4.  **DEVELOPER PROFILE**: AIにあなたの開発スタイルや好み（例: 「Tailwind CSSを使用」「日本語コメントを多用」「モダンなダークモードデザインを好む」）を記述します。AIの生成結果に影響を与えます。
5.  **CHAT MODEL / CODE MODEL**: 使用したいGeminiモデルをそれぞれ選択します。APIキーが設定されている場合のみ、より高性能なモデルを選択できます。
6.  全ての入力が完了したら、**「設定を保存」ボタン**をクリックして変更を適用します。

#### 2. 要件定義とプランニング（CHATタブ）

1.  **「CHAT」タブ**をクリックします。
2.  下部の入力エリアに、作成したいWebサイトやアプリの要件を日本語で入力します（例: 「ダークモードで動作するシンプルなTodoアプリを作りたい」）。
3.  必要に応じて、入力エリア左横の**「＋」ボタン**をクリックし、画像ファイル（カメラ撮影、ギャラリーから選択、ファイル添付）をアップロードしてUIイメージなどをAIに伝えます。
4.  **「↑」送信ボタン**をクリックしてAIとの対話を開始します。
5.  AIが提案するプランニングに沿って対話を続け、要件を具体化します。
6.  プランが固まると、AIが「素晴らしいプランです。⚡️コードを作成 ボタンから実装を開始しましょう」とメッセージを表示します。その指示に従い**「⚡️コードを作成 (BUILD)」ボタン**をクリックします。

#### 3. コード生成と編集（CODEタブ）

1.  AIがコードを生成し始めると、自動的に**「CODE」タブ**に切り替わるか、バックグラウンド生成通知が表示されます。
2.  CodeMirrorエディタに生成されたHTML/CSS/JavaScriptコードが表示されます。
3.  必要に応じてコードを手動で修正・改善できます。
    *   **📥 Import**: GitHubリポジトリからファイルを選択し、エディタにインポートします。
    *   **Upload**: ローカルのコードファイルをアップロードし、エディタに読み込みます。
    *   **🔍 Search**: エディタ内のコードを検索します。
    *   **Download**: 現在のエディタのコードを `index.html` としてダウンロードします。
    *   **🚀 GitHub Deploy**: 現在のコードをGitHubリポジトリにデプロイします（次項を参照）。

#### 4. リアルタイムプレビューとデバッグ（PREVIEWタブ）

1.  **「PREVIEW」タブ**をクリックすると、現在のエディタのコードがブラウザにリアルタイムで表示されます。
2.  下部のデバッグパネルにある入力エリアに、プレビューで気になった点や修正したい内容を入力します（例: 「ボタンの色を青から緑に変更してほしい。また、フォントサイズを少し大きくしてほしい」）。
3.  **「修正を依頼」ボタン**をクリックすると、その内容が「CHAT」タブのAIに送信され、修正プランニングとコード生成が再開されます。

#### 5. GitHub Deploy（CODEタブから）

1.  **「CODE」タブ**でコードが完成したら、右上の**「🚀 GitHub Deploy」ボタン**をクリックします。
2.  **GitHubトークン設定**: 未設定の場合は、GitHub Personal Access Tokenを入力して認証します。
3.  **リポジトリ選択**: 既存のGitHubリポジトリを選択するか、**「＋ 新規リポジトリを作成」ボタン**で新しいリポジトリを作成します。
4.  **デプロイ詳細**:
    *   デプロイ先のブランチ（例: `main`）とファイルパス（例: `index.html`）を指定します。
    *   AIが生成したプロジェクトのREADME.mdの内容が自動で表示されるので、必要に応じて修正できます。
    *   **「GitHub Pagesを有効にして公開する」**チェックボックスにチェックを入れると、コードをプッシュ後にGitHub Pagesが自動的に有効化され、Webサイトとして公開されます。
5.  **「🚀 デプロイ実行」ボタン**をクリックすると、指定したリポジトリにコードがプッシュされます。
